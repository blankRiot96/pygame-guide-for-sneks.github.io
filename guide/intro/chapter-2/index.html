In this article, we'll go over how to get a basic window running.

Table of contents

Setup

Create a file, for the purposes of this article I'll be calling it main.py. Avoid calling it pygame.py as that would result in a circular import error.

Now obviously, the first thing to do is to import the library {% highlight python %} import pygame {% endhighlight %}

As a general practice, we generally initialize pygame before using it. This enables the use of audio, fonts, and other cool stuff that we don't have to remember to initialize individually {% highlight python %} import pygame pygame.init() {% endhighlight %}

Now here comes the interesting stuff, we're gonna spawn a window! {% highlight python %} import pygame pygame.init() pygame.display.set_mode(size=(500, 500)) {% endhighlight %} You probably didn't see it, but that opened a 500x500 (width x height) window and closed since the program terminated.

In order to keep the window open, the Python process needs to be running. Let's make a while loop to keep it running
CAUTION: Make sure you know how to end your Python program forcefully, as the Window will be unresponsive. If you're running this from a terminal, using CTRL+C or CTRL+SHIFT+C will do {% highlight python %} import pygame pygame.init() pygame.display.set_mode(size=(500, 500)) while True: pass {% endhighlight %}

Great! Now we've got a black window running, but it's unresponsive. Let's fix that, shall we? {% highlight python %} import pygame pygame.init() pygame.display.set_mode(size=(500, 500)) while True: events = pygame.event.get() for event in events: pass {% endhighlight %}
events = pygame.event.get(): Here, we're requesting the events that happened since the previous time pygame.event.get was called. So any keypress, mouse input or touch input was recorded here.

In this case, we want to loop over these events and check if the user pressed the X button. pygame.QUIT is sent when the user closes the window {% highlight python %} import pygame pygame.init() pygame.display.set_mode(size=(500, 500)) while True: events = pygame.event.get() for event in events: if event.type == pygame.QUIT: pass {% endhighlight %}
if event.type == pygame.QUIT: So the .type attribute of each event is how we determine what the type of each event is. This maps to an integer constant and is usually compared with a named constant within pygame like .QUIT {% highlight python %} import pygame pygame.init() pygame.display.set_mode(size=(500, 500)) while True: events = pygame.event.get() for event in events: if event.type == pygame.QUIT: raise SystemExit {% endhighlight %}
raise SystemExit: Well, this is pretty self explanatory. This simply raises a system exit, which terminates the Python program, and as we know - that causes the window to close


Now what if we want to draw something onto the window?
In order to do that, let's first get the display Surface.
A surface is basically a canvas we can draw pixels on, and the display surface is the main canvas which represents the area on our window.

The display surface is returned by the display.set_mode function, let's assign that to a variable. {% highlight python %} import pygame pygame.init() screen = pygame.display.set_mode(size=(500, 500)) while True: events = pygame.event.get() for event in events: if event.type == pygame.QUIT: raise SystemExit {% endhighlight %} In pygame, it is common to name this variable 'screen'. Let's go ahead and fill the display surface, i.e, 'screen' with a color. {% highlight python %} import pygame pygame.init() screen = pygame.display.set_mode(size=(500, 500)) while True: events = pygame.event.get() for event in events: if event.type == pygame.QUIT: raise SystemExit screen.fill("blue") {% endhighlight %} Surface.fill lets you fill the entire surface/canvas with a color of your choice.

But hey, this didn't do anything? The window is still black!
The reason for this is that we need to tell the window to update itself each time we draw something onto it. We can do that with pygame.display.flip() {% highlight python %} import pygame pygame.init() screen = pygame.display.set_mode(size=(500, 500)) while True: events = pygame.event.get() for event in events: if event.type == pygame.QUIT: raise SystemExit screen.fill("blue") pygame.display.flip() {% endhighlight %} And voila! We have a perfectly good pygame window running
